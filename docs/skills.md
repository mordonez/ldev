# AI Skills

`ldev` provides a documented, repeatable way to install and update reusable AI skills in Liferay projects.

This page is the canonical reference for skill setup and prompt usage.

`ldev ai install` is now mode-aware:

- in `ldev-native`, it installs `AGENTS.md`, `CLAUDE.md`, vendor skills, and optional project-owned context/overlays
- in `blade-workspace`, it treats the official Workspace AI files as the base layer and adds complementary `ldev-*` rule files plus the reusable vendor skills

## Install skills

Install vendor-managed skills into the current repo:

```bash
ldev ai install --target .
```

Install project overlays too (`project-*` skills and Claude agents):

```bash
ldev ai install --target . --project
```

Install project context scaffolding when the repo will maintain it:

```bash
ldev ai install --target . --project-context
```

This overlay is capability-aware. In `blade-workspace`, `ldev` installs only the project-owned pieces that make sense for that runtime and warns when native-only overlays are intentionally skipped.

Install only a selected subset of vendor skills:

```bash
ldev ai install --target . --skill liferay-expert --skill developing-liferay
```

## Update skills

Refresh vendor-managed skills listed in `.agents/.vendor-skills`:

```bash
ldev ai update --target .
```

Re-scope vendor-managed skills to a selected set:

```bash
ldev ai update --target . --skill liferay-expert
```

## What gets installed

- `.agents/skills/<skill-name>/SKILL.md` for vendor-managed skills.
- `.agents/.vendor-skills` manifest generated by `ldev ai`.
- `AGENTS.md` bootstrap entrypoint (on install).
- `CLAUDE.md` in `ldev-native`.
- Optional project context scaffolding with `--project-context` or `--project`:
  - `docs/ai/project-context.md`
  - `docs/ai/project-context.md.sample`
- Optional project overlays with `--project`:
  - `.agents/skills/project-*`
  - `.claude/agents/*.md`

Additional behavior in `blade-workspace`:

- install `ldev-*` rule files under `.workspace-rules/`
- fan out those same `ldev-*` files to:
  - `.claude/rules/`
  - `.cursor/rules/`
  - `.gemini/`
  - `.github/instructions/`
  - `.windsurf/rules/`
- preserve official Workspace files such as:
  - `.workspace-rules/liferay-rules.md`
  - `.claude/CLAUDE.md`
  - `.github/copilot-instructions.md`

## Prompt usage examples

Use explicit skill routing in prompts for more reliable outcomes.

### 1) Router prompt

```text
Use liferay-expert.
Task: "I need to export a structure, review dependencies, and prepare a safe import into another environment."
```

### 2) Implementation prompt

```text
Use developing-liferay.
Task: "Implement a fragment + ADT and return a plan with ldev commands and validation criteria."
```

### 3) Deployment prompt

```text
Use deploying-liferay.
Task: "Build and deploy the current change, verify OSGi state, and provide a rollback plan if it fails."
```

### 4) Troubleshooting prompt

```text
Use troubleshooting-liferay.
Task: "Portal is not starting: run a layered diagnosis and prioritize recovery actions."
```

### 5) Browser validation prompt

```text
Use automating-browser-tests.
Task: "Automate page-editor navigation, validate rendering, and generate visual evidence."
```

## Best practices

1. Start sessions with:

```bash
ldev doctor --json
ldev context --json
```

2. Keep vendor and project knowledge separated:
   - Vendor: `.agents/.vendor-skills` + standard skills.
   - Project-specific: `docs/ai/project-context.md`, `.agents/skills/project-*`, and repository-owned overlays.

3. In standard Workspaces, keep the official Liferay AI files as the base layer and treat `ldev` as the workflow augmentation layer.

4. For team rollouts, prefer selecting a small set of skills first (`--skill`) and expand once the workflow is stable.
